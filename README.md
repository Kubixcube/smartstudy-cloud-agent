# SmartStudy Cloud Agent

SmartStudy Cloud Agent is a cloud-native RAG tutor for lecture PDFs.

A user uploads a PDF to Google Cloud Storage. A Cloud Function automatically extracts the text, splits it into chunks, generates embeddings with Vertex AI, and stores them in MongoDB Atlas Vector Search. The user can then ask questions about the indexed documents using Gemini 2.5 Flash.

---

## 1. What is already implemented?

* PDF upload bucket on Google Cloud Storage.
* Cloud Function triggered automatically when a PDF is uploaded.
* PDF text extraction with `pypdf`.
* Text splitting with `langchain-text-splitters`.
* Embedding generation with Vertex AI / `gemini-embedding-001`.
* Storage in MongoDB Atlas Vector Search.
* Local RAG question answering with Gemini 2.5 Flash.
* Tutor-style prompt with source references.

## 3. Requirements

You need:

* Python 3.12
* Conda or another Python environment manager
* Google Cloud CLI: `gcloud`
* Access to the Google Cloud project
* Access to the MongoDB Atlas cluster
* The repository cloned locally

## 4. Setup the Python environment

From the repository root:

```powershell
conda create -n smartstudy python=3.12 -y
conda activate smartstudy
pip install -r requirements.txt
```

The root `requirements.txt` should contain:

```txt
google-genai
google-cloud-storage
pymongo
python-dotenv
pypdf
langchain-core
langchain-text-splitters
streamlit
functions-framework
```

## 5. Configure environment variables

Create a `.env` file at the root of the repo.

Use `.env.example` as a template:

```env
GOOGLE_CLOUD_PROJECT=your-gcp-project-id
GOOGLE_CLOUD_LOCATION=us-central1

GCS_BUCKET_NAME=your-gcs-bucket-name

MONGODB_URI=mongodb+srv://USER:PASSWORD@CLUSTER.mongodb.net/?retryWrites=true&w=majority
MONGODB_DB=smartstudy
MONGODB_COLLECTION=chunks
MONGODB_VECTOR_INDEX=vector_index
```

Never commit `.env`.

The `.gitignore` should include:

```gitignore
.env
cloud_function/env.yaml
__pycache__/
*.pyc
data/*.pdf
data/*.txt
```

## 7. MongoDB Atlas setup

The project expects the following MongoDB resources:

```text
Database: smartstudy
Collection: chunks
Vector Search Index: vector_index
```

The Vector Search index must use:

```text
Vector field: embedding
Dimensions: 3072
Similarity: cosine
```

JSON configuration:

```json
{
  "fields": [
    {
      "type": "vector",
      "path": "embedding",
      "numDimensions": 3072,
      "similarity": "cosine"
    },
    {
      "type": "filter",
      "path": "source_file"
    }
  ]
}
```

Choose **Bring your own embeddings** in MongoDB Atlas because embeddings are generated by Vertex AI, not by MongoDB.

## 8. Run local PDF ingestion

Put a PDF in the `data/` folder, for example:

```text
data/course.pdf
```

Run:

```powershell
python -m scripts.ingest_local data/course.pdf
```

Expected output:

```text
[1/4] Extracting text from: data\course.pdf
[2/4] Chunking X pages
[3/4] Generating embeddings for Y chunks
[4/4] Upserting chunks into MongoDB Atlas
Done. Upserted/modified documents: Y
```

Then check MongoDB Atlas. The `chunks` collection should contain documents with fields such as:

```text
text
source_file
page
chunk_index
chunk_id
embedding
updated_at
```

## 9. Ask questions locally

After at least one PDF has been ingested, run:

```powershell
python -m scripts.ask_local "What is this document about?"
```

Or ask a more specific question:

```powershell
python -m scripts.ask_local "Summarize the main concepts of this lecture."
```

Expected output:

```text
=== SmartStudy Answer ===
...

=== Retrieved Sources ===
- course.pdf, page 1, score=...
- course.pdf, page 2, score=...
```

Important: `ask_local.py` does not ingest PDFs. It only asks questions about documents that are already stored in MongoDB.

## 10. Deploy the Cloud Function

The Cloud Function automatically ingests PDFs uploaded to the GCS bucket.

### 10.1 Create `cloud_function/env.yaml`

Inside `cloud_function/`, create an `env.yaml` file:

```yaml
GOOGLE_CLOUD_PROJECT: "your-gcp-project-id"
GOOGLE_CLOUD_LOCATION: "us-central1"
MONGODB_URI: "mongodb+srv://USER:PASSWORD@CLUSTER.mongodb.net/?retryWrites=true&w=majority"
MONGODB_DB: "smartstudy"
MONGODB_COLLECTION: "chunks"
MONGODB_VECTOR_INDEX: "vector_index"
```

Never commit this file.

### 10.2 Cloud Function dependencies

`cloud_function/requirements.txt` should contain:

```txt
functions-framework==3.*
google-cloud-storage
google-genai
pymongo
pypdf
langchain-text-splitters
```

Do not add packages named `google` or `google-cloud`.

### 10.3 Deploy

From the repository root:

```powershell
$PROJECT_ID="your-gcp-project-id"
$REGION="us-central1"
$BUCKET_NAME="your-gcs-bucket-name"

gcloud config set project $PROJECT_ID
cd cloud_function
```

Deploy:

```powershell
gcloud functions deploy smartstudy-ingest `
  --gen2 `
  --runtime=python312 `
  --region=$REGION `
  --source=. `
  --entry-point=ingest_gcs_pdf `
  --trigger-bucket=$BUCKET_NAME `
  --memory=1024Mi `
  --timeout=540s `
  --env-vars-file=env.yaml
```

A successful deployment should show:

```text
state: ACTIVE
eventType: google.cloud.storage.object.v1.finalized
```

## 13. Security

Do not commit:

```text
.env
cloud_function/env.yaml
MongoDB passwords
Google credentials
PDF files
```

If a password is accidentally exposed:

1. Change it in MongoDB Atlas.
2. Update `.env`.
3. Update `cloud_function/env.yaml`.
4. Redeploy the Cloud Function.
